STA 141A: Predicting Wildfire Incidence in the United States¶

Overview¶

The goal of this final project is to develop a prediction model for the incidence and location of notable wildfires in the United States (link). You will use historical wildfire records to estimate the probability that a wildfire will occur in a given geographic area during a future month.

A full modeling pipeline is expected, including:

  • Careful problem formulation and data preprocessing
  • Exploratory analysis to motivate modeling choices
  • Predictive modeling with appropriate validation
  • Clear interpretation and communication of results

Students are expected to submit a final written report, organized according to the sections described below. Each student is expected to write their own reproducible report, but discussion is encouraged.

The goal of this project is to predict whether a notable wildfire will occur in a given state-month in the United States. To do this, I use historical wildfire records and transform the raw fire-level data into a dataset suitable for classification. The project includes data preprocessing, exploratory data analysis, predictive modeling, and time-aware validation. In particular, I compare a baseline logistic regression model with a decision tree model and evaluate their performance using AUC. The results show that wildfire incidence varies across both space and time, and that the decision tree provides better predictive performance than the baseline model.

Data and background¶

You can find and download a subset of historical wildfire records derived from the Kaggle dataset “188 Million US Wildfires”. The dataset includes wildfire ignition locations, discovery dates, containment dates, and final burned area across the contiguous United States. You are encouraged, but not required, to use the full dataset from the original source.

Wildfires vary widely in size, and the vast majority of recorded fires are very small and quickly contained. Modeling all recorded fires would lead to extreme class imbalance and would obscure the dynamics of fires that are operationally and scientifically meaningful.

To address this, you may filter the raw wildfire records to retain only “notable” fires. A recommended cutoff is to retain only the wildfires with final burned area $\geq$ 300 acres. This threshold is consistent with operational wildfire reporting standards used by U.S. fire management agencies, where large wildland fires are commonly defined as: $\geq$ 100 acres in timber fuel models, or $\geq$ 300 acres in grass or brush fuel models

Alternative Cutoffs. You may choose an alternative cutoff if you clearly justify your choice. Any alternative cutoff must be clearly justified in writing and applied consistently throughout the project.

I used 1000 acres instead of 300 acres to create a stricter definition of notable wildfire. This choice reduces the number of moderately sized fires included in the positive class and focuses the modeling task on larger wildfire events. Although 300 acres was the recommended cutoff, the alternative cutoff was applied consistently throughout the project.

Prediction Task¶

The prediction task is formulated as a binary classification problem with spatial and temporal structure.

  • Outcome
    For each spatial unit and month, define a binary variable indicating whether at least one notable wildfire occurs during that month.

  • Spatial unit
    You may choose an appropriate spatial unit (e.g., grid cells, counties, or other geographic partitions). The choice should balance spatial resolution, interpretability, and data sparsity.

  • Temporal unit
    Month. You may refine your resolution to week or day with clear justification.

  • Predictors
    You may select any variables in the dataset as your predictors. External data sources (e.g., weather or land cover) are allowed but not required.

spatial unit = STATE temporal unit = month outcome = whether at least one notable fire occurs predictors = total_fires, notable_fires, avg_fire_size, max_fire_size, month

I used state as the spatial unit because it provides a simple and interpretable geographic partition while avoiding excessive sparsity that could arise from using much finer spatial units.

The original wildfire dataset is recorded at the individual fire level, where each row corresponds to one wildfire event. However, the goal of this project is to predict whether a notable wildfire occurs in a given place and time period, so the raw data must be transformed into a modeling dataset that matches this prediction task. To do this, I converted the fire-level data into a state-by-month dataset, where each row represents one state in one month. This makes the final dataset consistent with the prediction problem and also makes the data preparation process reproducible.

Notable-fire cutoff: To identify important wildfire events, I created a binary variable called notable_fire. A fire was classified as notable if its FIRE_SIZE was at least 1000 acres. （FIRE_SIZE >= 1000） I chose this cutoff because it separates relatively large fires from small routine fires and gives a simple rule that is easy to explain and reproduce. This cutoff also fits the project goal, since the model is intended to predict larger and more meaningful wildfire events rather than every fire in the raw dataset. Time space: After defining notable fires, I created the time variables from the original DISCOVERY_DATE field. Since DISCOVERY_DATE was stored numerically, I converted it to a standard date format and then extracted the year and month. For the spatial unit, I used STATE. Combining these variables gave me state-month units, so each final observation corresponds to one state in one month. I used state-month rather than individual fires because the prediction task is whether a notable fire occurs in a broader space–time unit, not whether a single fire is large. This directly satisfies the rubric requirement for construction of space–time units. Binary outcome variable explanation: The final response variable in the modeling dataset is outcome, a binary variable defined at the state-month level. For each state-month, I set outcome = 1 if there was at least one notable fire during that month, and outcome = 0 otherwise. This definition is consistent with the prediction task because it turns the wildfire problem into a classification problem: the model predicts whether a state-month experienced at least one notable wildfire event. This addresses the rubric item requiring a binary outcome variable that is clearly constructed and aligned with the modeling goal.

Predictors and aggregation: To build the final dataset, I grouped the fire-level data by STATE, year, and month, and then aggregated the observations within each state-month. I created summary variables including the total number of fires, number of notable fires, average fire size, and maximum fire size. These variables summarize wildfire activity in each unit and can be used as predictors or descriptive variables in later modeling. This aggregation step is important because it transforms the raw event-level dataset into a structured dataset suitable for prediction at the state-month level.

Missing data and edge cases: Before aggregation, I removed rows with missing values in key variables such as STATE, LATITUDE, LONGITUDE, FIRE_SIZE, and DISCOVERY_DATE. These variables are necessary for defining the spatial unit, measuring fire severity, and assigning each event to the correct month. Removing incomplete rows ensures that the final dataset is internally consistent and avoids errors in the later modeling process. In addition, this step makes the preprocessing reproducible because the handling of missing observations is explicitly stated.

Final modeling dataset: The final modeling dataset contains one row per state-month. Its main variables are STATE, year, month, total_fires, notable_fires, avg_fire_size, max_fire_size, and outcome. This final dataset is the one used for predictive modeling. It is clearly tied to the prediction task because the predictors and outcome are all defined at the same state-month level.

4. Model Fitting and Validation¶

This section evaluates how well the models fit historical data and how they generalize over time. You need to use a proper validation method that respect the temporal structure.
